USR-VS

a webserver for ligand-based virtual screening powered by ultrafast shape recognition techniques

What are USR and USRCAT

USR-VS is the first webserver for large-scale prospective virtual screening using USR [1] and USRCAT [2], two validated ultrafast ligand-based 3D shape matching methods.

USR compares the 3D shape of two molecules, whereas USRCAT compares not only the 3D shape but also the atom type. Both methods are invariant to spatial rotation and translation, so they do not require structural alignment. Their implementation at USR-VS is highly optimized, able to screen more than 50 million 3D conformers per second. The similarity score between two molecules is normalized to (0, 1], with 1 being the most similar and 0 being the most dissimilar.

How to run a virtual screen

To run a virtual screen, there are only three simple steps, illustrated in Figure 1:

  1. provide a single-molecule query file (file size within 50KB) in SDF format specifying the 3D geometry of the molecule,
  2. choose either USR or USRCAT as the ranking score (USR by default),
  3. press the submit button.

Provided as two example query molecules are fluspirilene (ZINC ID: 00537755) and vemurafenib (PubChem CID: 42611257).

Figure 1: Running a virtual screen in three steps highlighted in blue color.

How to obtain a SDF file

There are public databases for users to obtain a SDF file for their desired compounds.

  • ZINC is a free database of over 35 million purchasable compounds in 3D formats, including SDF.
  • PubChem contains more than 79 million unique structures.

Users can search these third-party websites for their desired molecules using the compound name or the SMILES string, and then download the SDF file for the structure.

How to interpret the result

Once a virtual screen is submitted, the user will be redirected to the result webpage with a unique URL that is only available to the user. Users are suggested to bookmark the result webpage if they want to browse the result at a later time.

Figure 2 shows the result webpage of using ZINC00537755.sdf as the query molecule and choosing USR as the ranking score. In this webpage, a table at the top provides a link to the input file for download and shows the results of the virtual screen, including the submission time, execution time, completion time, and screening speed (this comes from dividing the 94 million 3D conformers by the time taken to compute their similarity scores to the query molecule, sort the scores and write the top hits).

Figure 2: Visualizing the query and hit molecules.

When the virtual screen is completed, the top 100 most similar molecules to the query molecule and their similarity scores are written to two output files for download (hits.csv and hits.sdf). Regardless of the selection of the ranking score, both USR and USRCAT similarities are calculated for the top hits.

By using the WebGL visualizer iview [3], the query molecule is shown in the left canvas, and the hit molecules are shown in the right canvas (this take may a few seconds to load, depending on the internet bandwidth). The user can switch among the top 100 hit molecules (named from 0 to 99) by pressing the button below the right canvas, and interactively translate, rotate and zoom in/out the 3D structure of the selected hit molecule to match the orientation of the query molecule. The latter permits to assess the degree of 3D similarity of both molecules. Note that neither USR nor USRCAT performs structural alignment, so the user has to manually align the hit molecule to the query molecule in order to intuitively inspect their similarity.

Also displayed are USR and USRCAT scores, chemical properties, and different options to purchase the hit molecule. This stage is intended to help the user decide which hits to purchase and how to purchase them to experimentally measure their activity against selected targets of the query molecule. Note that these targets can be both molecular (e.g. a protein of a known pathway) and non-molecular (a cancer cell line).

In this example run, the first hit molecule shown in the right canvas is identical to the query molecule shown in the left canvas, because the query molecule is one of the 23 million molecules of the screening library. This demonstrates USR's capability of correctly retriving an existing molecule from the database, thus validating the method. Such validity also applies to USRCAT, as can be seen in the corresponding result webpage. Note that the similarity scores are close to 1 but not exactly 1 because of floating point rounding error.

For the second hit molecule, there is a big difference between the USR result webpage and the USRCAT result webpage. In the case of USR, the second hit molecule has a similar shape to the query molecule, regardless of their atom types. In the case of USRCAT, the second hit molecule is similar to the query molecule in terms of both shape and atom type.
